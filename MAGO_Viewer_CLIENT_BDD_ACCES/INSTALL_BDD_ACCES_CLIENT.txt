MAGO VIEWER - CLEAN WINDOWS INSTALLATION
========================================

Public repository policy
------------------------
The repository contains code and an empty database schema only.
It does not contain models, attributes, client accounts, passwords, GLB/PLY/SPZ
files, point clouds, Gaussian splats, PostgreSQL dumps or a real .env file.

Installation
------------
1. Clone or extract the repository.
2. Right-click INSTALLER_VIEWER_COMPLET.ps1 and run it with PowerShell, or run:

   powershell -NoProfile -ExecutionPolicy Bypass -File .\INSTALLER_VIEWER_COMPLET.ps1

3. Enter the local PostgreSQL password when prompted.
4. The installer creates empty databases mago_enrichment and mago_access,
   installs Node dependencies, builds the viewer, creates a local .env and a
   desktop shortcut, then tests /api/health.

Database contents created by the installer
------------------------------------------
mago_enrichment:
  mago_class, model, object, attribute, f_instantiate_object

mago_access:
  client_access and client_access_admin

Classes are created lazily from class keys found in imported MAGO scenes.
No example project or survey is installed.

Never commit
------------
.env, storage/, SAUVEGARDES_BDD/, *.dump, node_modules/, logs and client URLs.
